# 教程：构建每日简报机器人

在本教程中，你将构建一个个人简报机器人，它每天早上醒来，研究你关心的主题，总结发现，并将一份简洁的简报直接发送到你的 Telegram 或 Discord。

最终，你将拥有一个结合了**网络搜索**、**定时任务调度**、**委派**和**消息传递**的完全自动化工作流——无需编写代码。

## 我们要构建什么

流程如下：

1.  **上午 8:00** — 定时任务调度器触发你的任务
2.  **Hermes 启动**一个带有你提示词的全新 Agent 会话
3.  **网络搜索**获取你关注主题的最新消息
4.  **摘要总结**将其提炼成简洁的简报格式
5.  **投递**将简报发送到你的 Telegram 或 Discord

整个过程自动运行，无需手动操作。你只需在喝早咖啡时阅读简报。

## 先决条件

开始之前，请确保你已具备：

-   **已安装 Hermes Agent** — 参见[安装指南](/docs/getting-started/installation)
-   **消息网关正在运行** — 消息网关守护进程处理定时任务执行：
    ```bash
    hermes gateway install   # 安装为用户服务
    sudo hermes gateway install --system   # Linux 服务器：开机启动的系统服务
    # 或者
    hermes gateway           # 在前台运行
    ```
-   **Firecrawl API 密钥** — 在环境中设置 `FIRECRAWL_API_KEY` 以进行网络搜索
-   **已配置消息传递**（可选但推荐） — 已设置好 [Telegram](/docs/user-guide/messaging/telegram) 或 Discord 并配置了主频道

:::tip 没有消息传递？没问题
你仍然可以使用 `deliver: "local"` 来学习本教程。简报将保存到 `~/.hermes/cron/output/`，你可以随时阅读。
:::

## 步骤 1：手动测试工作流

在自动化任何操作之前，先确保简报功能正常。启动一个聊天会话：

```bash
hermes
```

然后输入以下提示词：

```
搜索关于 AI Agent 和开源 LLM 的最新消息。
以简洁的简报格式总结前 3 个故事，并附上链接。
```

Hermes 将搜索网络，浏览结果，并生成类似以下的内容：

```
☀️ 你的 AI 简报 — 2026年3月8日

1. Qwen 3 发布，拥有 2350 亿参数
   阿里巴巴最新的开源权重模型在多个基准测试中与 GPT-4.5 匹敌，同时保持完全开源。
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain 推出 Agent 协议标准
   一个新的 Agent 间通信开放标准在发布第一周就获得了 15 个主要框架的采用。
   → https://blog.langchain.dev/agent-protocol/

3. 欧盟《人工智能法案》开始对通用模型强制执行
   首批合规截止日期到来，开源模型在 1000 万参数阈值以下获得豁免。
   → https://artificialintelligenceact.eu/updates/

---
3 个故事 • 搜索了 8 个来源 • 由 Hermes Agent 生成
```

如果这能正常工作，你就可以准备将其自动化了。

:::tip 迭代优化格式
尝试不同的提示词，直到获得你满意的输出。添加诸如“使用表情符号标题”或“每个摘要保持在 2 句话以内”的指令。你最终确定的提示词将用于定时任务。
:::

## 步骤 2：创建定时任务

现在让我们安排这个任务每天早上自动运行。你可以通过两种方式实现。

在创建定时任务之前，请确保 Hermes 已在全局配置了默认的模型和提供商。如果你希望某个特定任务使用不同的值，请在创建时设置明确的任务级模型/提供商覆盖。

### 选项 A：自然语言（在聊天中）

直接告诉 Hermes 你想要什么：

```
每天早上 8 点，搜索关于 AI Agent 和开源 LLM 的最新消息。
以简洁的简报格式总结前 3 个故事，并附上链接。使用友好、专业的语气。投递到 telegram。
```

Hermes 将使用统一的 `cronjob` 工具为你创建定时任务。

### 选项 B：CLI 斜杠命令

使用 `/cron` 命令以获得更多控制：

```
/cron add "0 8 * * *" "搜索关于 AI Agent 和开源 LLM 的最新消息。查找过去 24 小时内至少 5 篇近期文章。以简洁的每日简报格式总结前 3 个最重要的故事。对于每个故事，请包含：清晰的标题、2 句话的摘要和来源 URL。使用友好、专业的语气。使用表情符号项目符号进行格式化，并以故事总数结尾。"
```

### 黄金法则：自包含的提示词

:::warning 关键概念
定时任务在**完全全新的会话**中运行——没有之前对话的记忆，没有关于你“之前设置”的上下文。你的提示词必须包含 Agent 执行任务所需的**所有**信息。
:::

**不好的提示词：**
```
做我通常的晨间简报。
```

**好的提示词：**
```
搜索关于 AI Agent 和开源 LLM 的最新消息。
查找过去 24 小时内至少 5 篇近期文章。以简洁的每日简报格式总结前 3 个最重要的故事。对于每个故事，请包含：清晰的标题、2 句话的摘要和来源 URL。使用友好、专业的语气。使用表情符号项目符号进行格式化。
```

好的提示词明确了**搜索什么**、**多少篇文章**、**什么格式**以及**什么语气**。它包含了 Agent 一次性完成任务所需的一切。

## 步骤 3：定制简报

一旦基本简报正常工作，你就可以发挥创意了。

### 多主题简报

在一个简报中涵盖多个领域：

```
/cron add "0 8 * * *" "创建一个涵盖三个主题的晨间简报。对于每个主题，搜索过去 24 小时内的近期消息，并总结前 2 个故事及其链接。

主题：
1. 人工智能和机器学习 — 关注开源模型和 Agent 框架
2. 加密货币 — 关注比特币、以太坊和监管新闻
3. 太空探索 — 关注 SpaceX、NASA 和商业航天

格式化为带有章节标题和表情符号的简洁简报。以今天的日期和一句励志名言结尾。"
```

### 使用委派进行并行研究

为了更快地生成简报，告诉 Hermes 将每个主题委派给一个子 Agent：

```
/cron add "0 8 * * *" "通过将研究委派给子 Agent 来创建晨间简报。委派三个并行任务：

1. 委派：搜索过去 24 小时内前 2 个 AI/ML 新闻故事及其链接
2. 委派：搜索过去 24 小时内前 2 个加密货币新闻故事及其链接
3. 委派：搜索过去 24 小时内前 2 个太空探索新闻故事及其链接

收集所有结果，并将其合并成一个带有章节标题、表情符号格式化和来源链接的简洁简报。添加今天的日期作为标题。"
```

每个子 Agent 独立且并行地进行搜索，然后主 Agent 将所有内容合并成一个精炼的简报。有关其工作原理的更多信息，请参阅[委派文档](/docs/user-guide/features/delegation)。

### 仅限工作日的时间表

周末不需要简报？使用针对周一至周五的 cron 表达式：

```
/cron add "0 8 * * 1-5" "搜索最新的 AI 和科技新闻..."
```

### 每日两次简报

获取早间概览和晚间回顾：

```
/cron add "0 8 * * *" "早间简报：搜索过去 12 小时内的 AI 新闻..."
/cron add "0 18 * * *" "晚间回顾：搜索过去 12 小时内的 AI 新闻..."
```

### 使用记忆添加个人上下文

如果你启用了[记忆](/docs/user-guide/features/memory)，你可以存储跨会话持久化的偏好设置。但请记住——定时任务在没有对话记忆的全新会话中运行。要添加个人上下文，请直接将其嵌入到提示词中：

```
/cron add "0 8 * * *" "你正在为一位关心以下方面的高级 ML 工程师创建简报：PyTorch 生态系统、Transformer 架构、开源权重模型以及欧盟的 AI 监管。跳过关于产品发布或融资轮次的报道，除非它们涉及开源。

搜索这些主题的最新消息。总结前 3 个故事及其链接。保持简洁和技术性——这位读者不需要基础解释。"
```

:::tip 定制角色
包含关于简报是*为谁*制作的详细信息，可以显著提高相关性。告诉 Agent 你的角色、兴趣以及要跳过什么。
:::

## 步骤 4：管理你的任务

### 列出所有已安排的任务

在聊天中：
```
/cron list
```

或者在终端中：
```bash
hermes cron list
```

你将看到类似以下的输出：

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### 删除任务

在聊天中：
```
/cron remove a1b2c3d4
```

或者以对话方式询问：
```
删除我的晨间简报定时任务。
```

Hermes 将使用 `cronjob(action="list")` 来查找它，并使用 `cronjob(action="remove")` 来删除它。

### 检查消息网关状态

确保调度程序实际在运行：

```bash
hermes cron status
```

如果消息网关没有运行，你的任务将不会执行。为了可靠性，将其安装为后台服务：

```bash
hermes gateway install
# 或者在 Linux 服务器上
sudo hermes gateway install --system
```

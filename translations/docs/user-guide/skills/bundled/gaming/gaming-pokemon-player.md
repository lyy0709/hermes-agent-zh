# Pokemon Player

使用 `pokemon-agent` 包通过无头模拟玩宝可梦游戏。

## 使用时机
- 用户说"玩宝可梦"、"开始宝可梦"、"宝可梦游戏"
- 用户询问关于宝可梦红、蓝、黄、火红等版本
- 用户想观看 AI 玩宝可梦
- 用户提及 ROM 文件 (.gb, .gbc, .gba)

## 启动流程

### 1. 首次设置（克隆、虚拟环境、安装）
仓库是 GitHub 上的 NousResearch/pokemon-agent。克隆它，然后
设置 Python 3.10+ 虚拟环境。使用 uv（推荐，速度更快）
创建虚拟环境并以可编辑模式安装带有 pyboy 额外依赖的包。
如果 uv 不可用，则回退到 python3 -m venv + pip。

在本机上，它已设置在 /home/teknium/pokemon-agent
并且虚拟环境已准备就绪 — 只需 cd 到该目录并 source .venv/bin/activate。

你还需要一个 ROM 文件。向用户询问他们的 ROM 文件。在本机上，
该目录内的 roms/pokemon_red.gb 处已存在一个。
**切勿**下载或提供 ROM 文件 — 始终询问用户。

### 2. 启动游戏服务器
在 pokemon-agent 目录内，激活虚拟环境后，运行
pokemon-agent serve，并使用 --rom 指向 ROM 文件，--port 9876。
使用 & 在后台运行它。
要从保存的游戏恢复，请添加 --load-state 并指定保存名称。
等待 4 秒启动，然后使用 GET /health 验证。

### 3. 为用户设置实时仪表板以观看
通过 localhost.run 使用 SSH 反向隧道，以便用户可以在
他们的浏览器中查看仪表板。使用 ssh 连接，将本地
端口 9876 转发到 nokey@localhost.run 的远程端口 80。将输出
重定向到日志文件，等待 10 秒，然后 grep 日志查找 .lhr.life
URL。给用户该 URL，并在末尾加上 /dashboard/。
隧道 URL 每次都会变化 — 如果重新启动，请给用户新的 URL。

## 保存与加载

### 何时保存
- 每进行 15-20 回合游戏
- **始终**在道馆战、劲敌遭遇或高风险战斗之前
- 进入新城镇或地牢之前
- 在任何你不确定的行动之前

### 如何保存
POST /save 并附带描述性名称。好的例子：
before_brock, route1_start, mt_moon_entrance, got_cut

### 如何加载
POST /load 并附带保存名称。

### 列出可用保存
GET /saves 返回所有保存的状态。

### 在服务器启动时加载
启动服务器时使用 --load-state 标志以自动加载保存。
这比启动后通过 API 加载更快。

## 动作参考
- press_a — 确认、交谈、选择
- press_b — 取消、关闭菜单
- press_start — 打开游戏菜单
- walk_up/down/left/right — 移动一格
- hold_b_N — 按住 B 键 N 帧（用于快速跳过文本）
- wait_60 — 等待约 1 秒（60 帧）
- a_until_dialog_end — 反复按 A 直到对话框清除

## 来自经验的关键提示

### 持续使用视觉
- 每移动 2-4 步就截图一次
- RAM 状态告诉你位置和 HP，但**不**告诉你周围有什么
- 岩壁、栅栏、标志、建筑门、NPC — 只能通过截图看到
- 向视觉模型提出具体问题："我北边一格是什么？"
- 卡住时，在尝试随机方向前务必截图

### 传送过渡需要额外等待时间
当走过门或楼梯时，屏幕在地图过渡期间会淡出为黑色。
你**必须**等待它完成。在任何门/楼梯传送后添加 2-3 个 wait_60
动作。不等待的话，位置读取会过时，你会以为还在旧地图。

### 建筑出口陷阱
当你离开建筑时，你会直接出现在门**前面**。
如果你向北走，你会直接回到里面。**始终**先侧移
向左或向右走 2 格，然后再按你打算的方向前进。

### 对话框处理
第一世代的文本是逐个字母缓慢滚动的。要快速跳过对话框，
按住 B 键 120 帧然后按 A。根据需要重复。按住 B 键使
文本以最快速度显示。然后按 A 键前进到下一行。
a_until_dialog_end 动作检查 RAM 对话框标志，但这个标志
并不能捕获**所有**文本状态。如果对话框似乎卡住了，请改用手动
hold_b + press_a 模式，并通过截图验证。

### 岩壁是单向的
岩壁（小悬崖边缘）只能向下（南）跳，永远不能向上（北）爬。
如果被北边的岩壁挡住，你必须向左或向右走以找到
绕过它的缺口。使用视觉来识别缺口在哪个方向。
明确询问视觉模型。

### 导航策略
- 一次移动 2-4 步，然后截图检查位置
- 进入新区域时，立即截图以定位
- 询问视觉模型"去 [目的地] 走哪个方向？"
- 如果卡住超过 3 次尝试，截图并完全重新评估
- 不要连续发送 10-15 个移动动作 — 你会冲过头或卡住

### 从野外战斗中逃跑
在战斗菜单上，逃跑在右下角。要从默认
光标位置（战斗，左上角）到达那里：按向下然后向右将光标
移动到逃跑，然后按 A。用 hold_b 包裹以快速跳过文本/动画。

### 战斗（战斗）
在战斗菜单上，战斗在左上角（默认光标位置）。
按 A 进入招式选择，再按 A 使用第一个招式。
然后按住 B 以快速跳过攻击动画和文本。

## 战斗策略

### 决策树
1. 想捕捉？ → 削弱然后扔精灵球
2. 不需要的野生宝可梦？ → 逃跑
3. 有属性优势？ → 使用效果绝佳的招式
4. 没有优势？ → 使用最强的本系招式
5. HP 低？ → 切换或使用伤药

### 第一世代属性相克表（关键对阵）
- 水克火、地面、岩石
- 火克草、虫、冰
- 草克水、地面、岩石
- 电克水、飞行
- 地面克火、电、岩石、毒
- 超能力克格斗、毒（在第一世代占主导地位！）

### 第一世代特性
- 特殊能力值 = 特殊招式的攻击**和**防御
- 超能力属性过于强大（幽灵招式有 Bug）
- 要害一击基于速度能力值
- 缠绕/捆绑阻止对手行动
- 聚气 Bug：**降低**要害率而不是提高

## 停止游戏
1. 通过 POST /save 使用描述性名称保存游戏
2. 使用 PKM:PROGRESS 更新记忆
3. 告诉用户："游戏已保存为 [名称]！说'玩宝可梦'以继续。"
4. 终止服务器和隧道后台进程

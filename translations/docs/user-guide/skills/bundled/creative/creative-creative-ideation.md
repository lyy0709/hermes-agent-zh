# 创意构思

## 使用时机

当用户说“我想做点什么”、“给我一个项目点子”、“我好无聊”、“我该做什么”、“给我点灵感”，或任何“我有工具但没有方向”的变体时使用。适用于代码、艺术、硬件、写作、工具以及任何可以制作的东西。

通过创意约束生成项目想法。约束 + 方向 = 创造力。

## 工作原理

1.  **选择一个约束** — 从下面的库中随机选择，或根据用户的领域/心情匹配
2.  **广泛地解释它** — 一个编码提示可以变成一个硬件项目，一个艺术提示可以变成一个 CLI 工具
3.  **生成 3 个满足约束的具体项目想法**
4.  **如果他们选择了一个，就构建它** — 创建项目，编写代码，交付成果

## 规则

每个提示都应尽可能广泛地解释。“这包括 X 吗？” → 是的。提示提供方向和温和的约束。没有这两者，就没有创造力。

## 约束库

### 面向开发者

**解决你自己的痛点：**
构建你这周希望存在的工具。少于 50 行代码。今天就交付。

**自动化烦人的事情：**
你工作流程中最乏味的部分是什么？用脚本解决它。花两个小时解决一个每天浪费你五分钟的问题。

**那个应该存在的 CLI 工具：**
想想你希望可以输入的命令。`git undo-that-thing-i-just-did`。`docker why-is-this-broken`。`npm explain-yourself`。现在构建它。

**除了胶水，别无新意：**
完全使用现有的 API、库和数据集制作一些东西。唯一的原创贡献在于你如何连接它们。

**弗兰肯斯坦周：**
拿一个做 X 的东西，让它做 Y。一个播放音乐的 git 仓库。一个生成诗歌的 Dockerfile。一个发送赞美的定时任务。

**减法：**
你能从一个代码库中移除多少东西而不破坏它？将一个工具剥离到其最小可行功能。删除直到只剩下本质。

**高概念，低投入：**
一个深刻的想法，懒散地执行。概念应该很出色。实现应该花一个下午。如果花的时间更长，那就是你想太多了。

### 面向创客和艺术家

**明目张胆地复制某物：**
挑选你欣赏的东西 — 一个工具、一件艺术品、一个界面。从头开始重新创建它。学习就在你的版本和他们的版本之间的差距中。

**一百万个某物：**
一百万既是很多，也没那么多。一百万个像素是一张 1MB 的照片。一百万个 API 调用是一个星期二。任何东西达到一百万规模都会变得有趣。

**制作一个会消亡的东西：**
一个每天失去一个功能的网站。一个会忘记的聊天机器人。一个倒计时到虚无。一次关于腐烂、杀戮或放手的练习。

**做很多数学：**
生成几何、着色器高尔夫、数学艺术、计算折纸。是时候重新学习 arcsin 是什么了。

### 面向任何人

**文本是通用接口：**
构建一个文本是唯一界面的东西。没有按钮，没有图形，只有文字输入和输出。文本几乎可以进出任何东西。

**从笑点开始：**
想一个会是有趣句子的事情。反向工作使其成真。“我教我的恒温器对我进行煤气灯操纵” → 现在构建它。

**敌对 UI：**
制作一个故意难以使用的东西。一个需要满足 47 个条件的密码字段。一个每个标签都在撒谎的表单。一个评判你命令的 CLI。

**再来一次：**
回忆一个旧项目。从头再做一次。不看原始版本。看看你的思维方式发生了什么变化。

更多关于沟通、规模、哲学、转化等方面的 30 多个额外约束，请参见 `references/full-prompt-library.md`。

## 将约束与用户匹配

| 用户说 | 从以下选择 |
|-----------|-----------|
| “我想做点什么”（没有方向） | 随机 — 任何约束 |
| “我正在学习 [语言]” | 明目张胆地复制某物，自动化烦人的事情 |
| “我想要点奇怪的东西” | 敌对 UI，弗兰肯斯坦周，从笑点开始 |
| “我想要点有用的东西” | 解决你自己的痛点，那个应该存在的 CLI，自动化烦人的事情 |
| “我想要点美丽的东西” | 做很多数学，一百万个某物 |
| “我筋疲力尽了” | 高概念低投入，制作一个会消亡的东西 |
| “周末项目” | 除了胶水，别无新意，从笑点开始 |
| “我想要一个挑战” | 一百万个某物，减法，再来一次 |

## 输出格式

```
## 约束：[名称]
> [约束，一句话]

### 想法

1.  **[一句话描述]**
    [2-3 句话：你会构建什么以及为什么它有趣]
    ⏱ [周末 / 一周 / 一个月] • 🔧 [技术栈]

2.  **[一句话描述]**
    [2-3 句话]
    ⏱ ... • 🔧 ...

3.  **[一句话描述]**
    [2-3 句话]
    ⏱ ... • 🔧 ...
```

## 示例

```
## 约束：那个应该存在的 CLI 工具
> 想想你希望可以输入的命令。现在构建它。

### 想法

1.  **`git whatsup` — 显示你离开期间发生了什么**
    比较你上次活动的提交和 HEAD，并总结发生了什么变化，
    谁提交的，以及合并了哪些 PR。就像来自你仓库的晨会。
    ⏱ 周末 • 🔧 Python, GitPython, click

2.  **`explain 503` — 面向人类的 HTTP 状态码**
    管道输入任何状态码或错误消息，并获得带有常见原因和修复方法的
    通俗易懂的解释。从精选的数据库中提取，而非 LLM。
    ⏱ 周末 • 🔧 Rust 或 Go，静态数据集

3.  **`deps why <package>` — 为什么这个包在我的依赖树中**
    将一个传递依赖追溯到引入它的直接依赖。
    用一个命令回答“为什么我有 47 个 lodash 副本”。
    ⏱ 周末 • 🔧 Node.js, npm/yarn lockfile 解析
```

用户选择后，开始构建 — 创建项目，编写代码，迭代。
